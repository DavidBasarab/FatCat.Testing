Phase 1 — Remaining Primitives & Value Types
These are all structurally similar to what you've already built, so you'll get them done fast:

bool (if not done)
char — equality, BeUpperCase(), BeLowerCase(), BeDigit(), BeWhiteSpace(), BeLetterOrDigit(), BeLetter()
byte / sbyte — same as int assertions (comparison, range, sign)
short / ushort — same
long / ulong — same
float — comparison + BeApproximately(), BeNaN(), NotBeNaN() (no Be()/NotBe())
double — same as float
decimal — same as int (does support exact Be()/NotBe())


Phase 2 — Common BCL Struct Types
Still value types but slightly richer:

DateTime / DateTimeOffset — big surface area (before/after, within range, HaveYear(), HaveMonth(), HaveDay(), HaveHour(), etc.)
TimeSpan — comparison + BePositive(), BeNegative(), BeCloseTo()
Guid — Be(), NotBe(), BeEmpty(), NotBeEmpty()
Enum — Be(), HaveFlag(), NotHaveFlag(), BeDefined()


Phase 3 — Collections
Before user objects, because they're still BCL and your numeric/string assertions feed directly into them:

IEnumerable<T> — the big one
IList<T> / List<T>
IDictionary<TKey, TValue>
array (has some unique assertions)


Phase 4 — Reference Types & User Objects
Once collections are solid:

object — the base assertions (BeNull, BeSameAs, BeOfType, BeAssignableTo, BeEquivalentTo)
Exception — WithMessage(), WithInnerException<T>(), etc.
User-defined objects — BeEquivalentTo() with deep graph comparison (this is the hardest one by far)


Phase 5 — Advanced / Async

Action / Func<T> — Throw<T>(), NotThrow(), CompleteWithin()
Task / Task<T> — async throw/completion assertions
AssertionScope — batching multiple failures
